# pipeline_latency_benmarking

This project built a **real-time latency benchmarking tool for a multi-process, multi-threaded Linux pipeline** in C & Python on Raspberry Pi, comparing **inter-process latency** (POSIX shared-memory, Unix domain socket), **inter-thread latency** (SPSC-ring buffer), **scheduling latency** (`clock_nanosleep()` wakeup delay) across a timestamped event pipeline (`event_gen → capture → process → response`) under mulitple environments and constrains. The system supports **SCHED_OTHER/FIFO/RR scheduling**, priorities assginment, **CPU pinning**, `mlockall()` memory locking, stress-load injection, and controlled CPU governor setup to expose how **IPC design, scheduler behavior, CPU affinity, and power management affect p99/worst-case latency**. **Python test automation** runs test pipeline, change & restores system settings, computes min/avg/max/p99 statistics, and generates latency histograms, demonstrating embedded Linux skills in **low-level C, POSIX IPC, real-time scheduling, python test automation, and system-level understanding & debugging**.
